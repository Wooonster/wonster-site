## Problem and Motivation

### What problem does this paper address?

LLM agents that use external skill libraries must retrieve relevant skills at inference time on every step. The goal is to eliminate this dependency by training agents to internalize skills directly into model parameters, enabling fully zero-shot autonomous behavior without any retrieval at runtime.

### Why do existing methods fail?

Retrieval-augmented skill use suffers from three structural problems:
1. **Retrieval noise** — the wrong skill or a partially relevant skill can be fetched, polluting the agent's context.
2. **Token overhead** — skill documents expand the context window at every step, increasing latency and cost.
3. **No knowledge acquisition** — the model never truly learns the skill; it merely reads it each time. Competence remains contingent on the retrieval scaffold.

### Why does this problem matter?

Agents that require external skill retrieval at inference time cannot be deployed reliably in latency-sensitive, resource-constrained, or offline environments. True skill internalization is a prerequisite for self-sufficient autonomous agents.

## Technical Method

### Core Contribution

SKILL0 proposes a training-time curriculum that progressively withdraws skill context — starting from full skill provision and decaying to zero — so that agents internalize tool invocation and multi-turn task completion directly into model weights, enabling zero-shot deployment with no skill retrieval.

### Pipeline

1. **Skill representation:** Skills are categorized and rendered together with interaction history into compact visual contexts, keeping each step's context under $0.5\text{k}$ tokens.
2. **Curriculum design:** Training begins with the full skill context available to the agent (maximum support). A dynamic curriculum evaluates skill helpfulness and applies a **linearly decaying budget** that progressively withdraws skill information over training.
3. **RL training:** The agent is trained via in-context agentic reinforcement learning. Rewards are derived from task completion signals across multi-turn interactions.
4. **Inference:** At deployment, the agent operates fully **zero-shot** — no skill retrieval, no external library, no expanded context.

### What is Actually New

Prior skill-augmented agents treat skills as inference-time documents. SKILL0 treats skill provision as a *training scaffold to be removed*, borrowing the curriculum logic from educational scaffolding theory. The key assumption abandoned is that skill context must be present at inference time.

### Complexity

- Inference context: **< $0.5\text{k}$ tokens per step** (vs. potentially several thousand tokens with retrieval-augmented approaches)

## Critical Assessment

**Strengths:**
- The curriculum design is principled and well-motivated by both cognitive science (scaffolding) and RL theory.
- Maintaining < $0.5\text{k}$ tokens per step at inference is a practical and deployable efficiency target.
- Gains on both embodied (ALFWorld) and QA (SearchQA) benchmarks suggest some generality.

**Concerns:**
- The paper evaluates on two benchmarks; broader evaluation across more diverse agent tasks would strengthen the claim of general skill internalization.
- It is unclear how the method handles skills that are highly compositional or that require reasoning chains not present in training data.
- The linearly decaying budget is a design choice that likely requires careful tuning; sensitivity analysis is not described in available content.

---

## Synthesis

**TL;DR:** SKILL0 trains LLM agents to internalize skills into model weights using a progressively withdrawing curriculum, achieving +9.7% on ALFWorld and +6.6% on SearchQA while operating zero-shot at inference with under $0.5\text{k}$ tokens per step. The main caveat is limited benchmark diversity.

**Innovation Classification:** *Method advance* — introduces a novel training curriculum within the established RL-for-agents framework that enables a qualitatively different inference regime.

**Open Problems:**
1. How does skill internalization interact with continual learning — can new skills be added without catastrophic forgetting?
2. Does the approach scale to skills that require long multi-step reasoning chains not expressible in compact visual contexts?
3. What is the minimum training data requirement per skill for reliable internalization?
